# Contributing to SiMM
Many thanks for your interest in contributing to SiMM, any types of feedbacks are warmly welcomed!
Please open an issue firstly to explain your ideas, such as features, bugfixs, doc/tool changes and so no, then link your PR to the issue.

# Code of Conduct
The code of conduct is described in [`CODE_OF_CONDUCT.md`](CODE_OF_CONDUCT.md)

## PR Title Format
Please use below unified prefix formats in your PR title:

- `[Feat]`  for new features  
- `[Fix]`   for bugfixes  
- `[Test]`  for test codes 
- `[Tool]`  for tools/utilities codes
- `[Build]` for build/ci issues 
- `[Doc]`   for documentation updates  
- `[Chore]` for any PRs that do not fit the above categories. But PLEASE use it prudentially.
